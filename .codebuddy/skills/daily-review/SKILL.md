# Daily Review

Help the user wrap up a day's work by pulling today's captured memories, organizing them into a structured review, and persisting the review back into dream-memory.

## When to trigger

Trigger when the user explicitly requests a wrap-up or review of today's work:

- "今天做了什么"
- "今天复盘一下"
- "日终总结"
- "wrap up today"
- "end of day"
- "今天的总结"

Do **NOT** trigger on:

- "今天天气怎么样" (chitchat)
- "今天的任务" (asking about tasks, not reviewing them)
- "review this code" (code review, different concept)

## Process

### Step 1: Recall today's memories

Call `dream_recall` to pull what was captured today. Use:

```
dream_recall({
  query: "<topic the user mentioned, OR a generic word like '工作' if no topic>",
  session_key: "<auto-generated by the MCP server's session manager — do NOT pass>"
})
```

If the user mentions a specific topic ("今天梦境系统的进展"), use that as the `query`. Otherwise use a broad query like `"今天"` or `"work"`.

### Step 2: Organize into 3-7 bullets

Group the recalled atoms into a short structured review. Example structure:

```
## 今天的工作回顾

**完成**：
- ...
- ...

**进行中**：
- ...

**决策**：
- ...

**遗留**：
- ...
```

Keep it tight. Aim for 3-7 bullets total. If `dream_recall` returns nothing useful, say `今天梦境记忆库里没有相关条目。要不要我先帮你回顾对话？` and offer to summarize from the recent conversation instead.

### Step 3: Show + ask for confirmation/补充

Show the review to the user. Ask:

> 这是今天的复盘草稿。要补充什么吗？补充完我会把最终版记入梦境。

Wait for the user. They may:

- Say "OK" / "可以" → proceed to Step 4
- Add more bullets → integrate, re-show, ask again
- Cancel → stop, don't capture

### Step 4: Capture the final review

Call `dream_capture_note` with:

- `text`: the full final review (the formatted markdown above)
- `metadata.tags`: `["daily-review", "<YYYY-MM-DD>"]`

### Step 5: Offer sublimation

After capturing, ask:

> 已记录。要现在触发一次升华吗（把今天的 L0 升华成 L1 atoms）？(y/n)

If yes, call `dream_session_end` with the auto-generated session_key.

If no, just say `OK，明天 23:00 cron 会自动同步触发升华。` and end.

## Failure handling

- `dream_recall` 失败：告知 Hermes 未启动，停止流程
- `dream_capture_note` 失败：告知用户复盘内容未持久化（建议复制到剪贴板备用）
- `dream_session_end` 失败：告知"升华失败但复盘已记入"，不重试

## What NOT to do

- ❌ Don't auto-trigger when the user just types "今天" without context
- ❌ Don't capture without showing the user first (Step 3 is mandatory)
- ❌ Don't generate fake bullets if `dream_recall` returns empty
- ❌ Don't loop: if user keeps adding 5+ rounds of additions, ask if they want to stop and just capture what's there

## Example flow

**User**: "今天复盘一下"

**You** (silently): Call `dream_recall({ query: "今天" })`

**You** (visible):
```
## 今天的工作回顾（2026-05-19）

**完成**：
- 阶段 3a：手工同步 758 chunks 入库
- 阶段 3b：cron-runner 实现，每晚 23:00 自动跑

**决策**：
- 阶段 3c 选 B 方案（mcp 接入 + 2 Skill）

**遗留**：
- DeepSeek API key 已暴露，待轮换

这是今天的复盘草稿。要补充什么吗？
```

**User**: "加一条：晚上验证了 cron real-run"

**You**: 整合后再展示，等用户确认。

**User**: "OK"

**You** (silently): Call `dream_capture_note(...)`, then ask about sublimation.
