## 3. Skill 设计要点

### capture-decision

**核心思路**：宁可漏不可错。description 里**显式写出反例**：

- "我在考虑 A 还是 B"（讨论中，不触发）
- "你觉得哪个好"（提问，不触发）
- "之前决定的那个 X"（回顾，不触发）

避免每次用户说"决定"二字就触发，污染记忆库。

**调用约定**：text 必须自包含一段话（不依赖对话上下文也能读懂）；alternatives 仅在对话里明确出现时才填。

### daily-review

**核心思路**：必须有人工 confirm gate。流程：recall → 整理 → 展示 → **等用户补充/确认** → capture → 询问升华。

**鲁棒设计**：

- recall 返回空 → 不编造 bullet，而是问用户是否要"从近期对话总结"
- 用户改 5+ 轮 → 主动问要不要停下捕获

## 4. mcp.json 配置选择

```json
"dream-memory": {
  "command": "node",
  "args": [
    "/Users/cmzhang/worker/job/workshop/dream-memory-mcp/dist/index.js",
    "--client-hint", "codebuddy"
  ]
}
```

**为什么不用 env**：原 examples 里是 `env.MCP_CLIENT_NAME`，但 dream-memory-mcp 的 CLI 实际是 `--client-hint`。统一为 args 风格（更显式可见）。

**为什么不显式 transport**：默认就是 stdio，CodeBuddy 也是 stdio 模式，省掉冗余。

**为什么不传 gateway-url**：默认 `http://localhost:8420` 就是对的，不用覆盖。
