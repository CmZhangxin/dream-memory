# 梦境系统设计文档

## 一、项目目标

构建一个个人梦境系统，让 AI 工具（CodeBuddy / Claude Code 等）的对话和编码经验能够**沉淀、关联、升华**为可复用的长期记忆，并能反哺给未来的对话。

**四个核心动机（递进关系）**：

1. **沉淀工作经验**——让 AI 做过的事不至于"做完就忘"，能被检索复用
2. **AI 懂我**——让任何 AI 工具上来就知道"我是谁、擅长什么、偏好什么"
3. **个人知识演化**——让笔记 + AI 对话自动聚合成"心智图谱"
4. **工程化兴趣**——产出可复用的 MCP 工具集，可开源

四个动机将通过**统一系统**实现，而非分裂成多个项目。

## 四、分阶段里程碑

```
阶段 1：地基                  阶段 2：通道                阶段 3：流入                阶段 4：呈现
─────────────                ─────────                   ─────────                   ─────────
Memory Gateway               MCP 工具集                  采集脚本                    Dashboard
跑起来                       (12 个工具)                 Skills 编排                 接入 + /persona

"能存数据了"                  "能写/读数据了"              "数据自动流入"              "看得见的梦境"
```

每个阶段独立可用，前一个不通过不开后一个。

### 阶段 1 · 地基（1 天）

- 在 `start.sh` / `stop.sh` 集成 Hermes Gateway 子进程
- 首次自动 clone TencentDB-Agent-Memory 到 `~/.dream-memory/`
- 配置 `.env.local` 的 LLM API Key
- **验收**：`./start.sh` 后 `curl localhost:8420/health` 返回 200

### 阶段 2 · 通道（2 天）

新建 `dream-memory-mcp/`，实现 12 个工具（清单见第六节）。

- 配置到 `~/.codebuddy/mcp.json`
- **验收**：CodeBuddy 对话中说"记下来：X"，能在 Memory 中查到

### 阶段 3 · 流入（2-3 天）

两条同步路径并行：

**A. 批处理脚本**（`dream-dashboard/scripts/`）：
- `sync-vault.ts`：增量扫描 Obsidian Vault
- `sync-codebuddy.ts`：扫描白名单项目的 `.codebuddy/memory/*.md`
- `daily-dream.ts`：定时入口（node-cron，跑在 Memory Gateway 进程里）

**B. CodeBuddy Skills**（`.codebuddy/skills/`）：
- `capture-decision`：沉淀重要决策
- `daily-review`：每日复盘流程
- `dream-now`：立即做一次梦

**验收**：每晚 23:00 自动同步前一日数据；CodeBuddy 中说"复盘今天"触发 Skill。

### 阶段 4 · 呈现（3-5 天）

dashboard 改造（详见第七节）：

- 新增 `/persona`（最简单）
- 改造 `/dream` `/brain` `/mission`
- 新增 4 个 API 代理路由

**验收**：`/persona` 页面显示出第一份画像（需阶段 3 跑满 3 天数据积累）。

**总工期**：约 8-12 个工作日。

## 五、数据流与采集策略

### 5.1 数据源映射

| 数据源 | 物理位置 | 进入层级 |
|--------|---------|---------|
| Obsidian Vault | `~/Documents/Obsidian Vault/**/*.md` | L0 → 自动升华 |
| CodeBuddy 项目日志（**主力**） | `<project>/.codebuddy/memory/YYYY-MM-DD.md` | L0 → 自动升华 |
| CodeBuddy 完整对话 | `~/.codebuddy/projects/<slug>/*.jsonl` | L0（敏感词过滤后） |
| CodeBuddy 历史记忆 | `~/.codebuddy/memery/*.md` | 直接导入 L3（首次 Bootstrap） |

### 5.2 三种采集方式

```
① 实时（Agent 触发）
   CodeBuddy 对话中调 dream_capture_* → 立即写入
   特点：高质量、低数量

② 增量（定时同步）
   daily-dream.ts 每晚 23:00 触发
   特点：普通质量、稳定数量

③ 全量（首次 Bootstrap）
   scripts/bootstrap.ts 一次性导入
   特点：一次性、大批量
```

### 5.3 CodeBuddy 数据源扫描决策

经实地探查 `~/.codebuddy/` 后确认：

| 数据源 | 价值 | 是否纳入 |
|-------|-----|---------|
| `~/.codebuddy/tasks/<uuid>/N.json` | 低（只是 todo 残留，无项目归属） | ❌ 不扫描 |
| `<project>/.codebuddy/memory/*.md` | **高**（结构化日报） | ✅ 主力 |
| `~/.codebuddy/projects/<slug>/*.jsonl` | 高（完整对话） | ✅ 第二优先 |

**项目识别方式**：白名单 `~/.dream-memory/projects.yml`

```yaml
projects:
  - path: /Users/cmzhang/worker/job/workshop
    name: workshop
    sources:
      - memory       # .codebuddy/memory/*.md
      - conversation # ~/.codebuddy/projects/<slug>/*.jsonl

  - path: /Users/cmzhang/worker/ai-rmall/ai-harness-coworker/backend
    name: rmall-backend
    sources: [memory, conversation]

ignore:
  - "**/passwords*"
  - "**/.env*"
  - "**/*-private.md"
```

**slug 反查算法**：
```typescript
function pathToSlug(path: string): string {
  return path.replace(/\//g, '-').replace(/^-/, '')
}
// /Users/cmzhang/worker/job/workshop → "Users-cmzhang-worker-job-workshop"
```

### 5.4 增量同步状态记录

`~/.dream-memory/sync-state.json`：

```json
{
  "vault": {
    "last_sync": "2026-05-18T23:00:00Z",
    "synced_files": { "path/to/note.md": "mtime_hash" }
  },
  "codebuddy_memory": {
    "last_sync": "2026-05-18T23:00:00Z",
    "synced_files": { "workshop/.codebuddy/memory/2026-05-18.md": "hash" }
  },
  "codebuddy_conversation": {
    "last_sync": "2026-05-18T23:00:00Z",
    "synced_files": { "Users-cmzhang-...jsonl:offset": 12345 }
  }
}
```

### 5.5 隐私边界

- 同步前正则扫描敏感词（密码 / token / 邮箱 / 手机号）
- 命中跳过 + 写日志，不进 Memory
- ignore 规则按 `.gitignore` 语法
- CodeBuddy 完整对话（projects/）默认参与同步，但触发更严格的敏感词过滤

### 5.6 定时任务实现

**方式**：独立的 node-cron 调度进程，跟 Hermes Gateway 同生命周期

- 不修改 Hermes Gateway（它是上游开源代码，不二改）
- 在 `dream-memory-mcp` 中提供一个 `cron-runner.ts` 入口
- `start.sh` 一并拉起，`stop.sh` 一并停止
- 不依赖 dashboard 是否打开
- 调度内容：每晚 23:00 调用同步工具 + `dream_dream_now`

## 六、MCP 工具集

共 **12 个工具**，命名前缀 `dream_*`。

### 6.1 写入类（5 个）

| # | 工具名 | 输入 | 用途 |
|---|--------|------|------|
| 1 | `dream_capture_note` | content, tags? | 通用：捕获任意文本到 L0 |
| 2 | `dream_capture_task` | title, description, status | 捕获任务/Todo |
| 3 | `dream_capture_conversation` | messages[], context? | 捕获完整对话片段 |
| 4 | `dream_capture_skill` | name, when, how, examples? | 直接沉淀到技能层（MVP 用 tag #skill 模拟） |
| 5 | `dream_capture_decision` | what, why, alternatives? | 沉淀重要决策（带理由） |

### 6.2 读取类（5 个）

| # | 工具名 | 输入 | 用途 |
|---|--------|------|------|
| 6 | `dream_search` | query, layer?, limit?, date_range? | 跨层级语义搜索 |
| 7 | `dream_get_persona` | — | 读取 L3 用户画像 |
| 8 | `dream_get_today` | — | 今日新增的 Atom / Scenario |
| 9 | `dream_get_scenarios` | active_only?, limit? | 列出活跃的 L2 场景 |
| 10 | `dream_get_related` | context, limit? | 基于上下文找相关记忆 |

### 6.3 触发类（2 个）

| # | 工具名 | 输入 | 用途 |
|---|--------|------|------|
| 11 | `dream_sync_source` | source ("vault" \| "codebuddy" \| "all") | 触发一次同步 |
| 12 | `dream_dream_now` | — | 立即触发 L0→L3 升华 |

### 6.4 调用关系

```
   Agent 启动             用户对话中触发            用户结束/定时
   ─────────              ─────────────             ─────────
   get_persona            capture_note              sync_source
   get_today              capture_skill             dream_now
                          search
                          get_related
                          get_scenarios

   "Agent 上线              "Agent 边聊边记            "异步整理升华"
    自我认知"                边查"                     做梦"
```

### 6.5 关键工具语义

**`dream_capture_skill`（MVP 策略）**：
- Memory Gateway 目前未完整实现 Procedural Memory 层
- MVP 阶段用 `dream_capture_note` + 特殊 tag `#skill` 模拟
- 等 Memory Gateway 支持后无缝升级

**`dream_dream_now` 的"做梦"语义**：
- 调用 Memory Gateway 的内部 pipeline 触发 L0→L1→L2→L3 升华
- 异步执行，立即返回 task_id
- 进度查询：`dream_search` 配合时间戳

## 七、Dashboard 改造（阶段 4）

### 7.1 页面规划

| 页面 | 状态 | 数据源 |
|------|------|------|
| `/dream` | 已有，改造 | `dream_get_today` |
| `/mission` | 已有，增强 | tasks + `dream_search` |
| `/brain` | 已有，改造 | `dream_get_scenarios` |
| `/persona` | **新增** | `dream_get_persona` |

### 7.2 API 代理层

dashboard 不直接调 Hermes Gateway，经过 Next.js 路由代理：

```
浏览器 → /api/memory/*  → Hermes Gateway :8420
         (Next.js Route)
```

**代理路由清单**：
- `GET /api/memory/today` → `dream_get_today`
- `GET /api/memory/persona` → `dream_get_persona`
- `GET /api/memory/scenarios` → `dream_get_scenarios`
- `GET /api/memory/search?q=xxx` → `dream_search`
- `GET /api/memory/health` → 健康检查 + 统计

### 7.3 UI 方向

- **`/dream`**：时间线风格，每个 Atom 一张卡片，带 tags 和关联 Scenario 链接
- **`/persona`**：四象限布局——兴趣 / 技能图谱 / 偏好 / 目标
- **`/brain`**：保留 force-graph，节点改为 L2 Scenario，悬停显示包含的 Atom
- **`/mission`**：现有列表 + 每行新增"关联记忆"链接

UI 细节在阶段 4 实施时迭代，本文档不约束像素级设计。

## 八、Skills 设计

### 8.1 三个 Skills（阶段 3 实现）

| Skill | 触发关键词 | 内部编排 |
|------|----------|---------|

```

├── daily-review/SKILL.md

```

遵循 CodeBuddy Skill 规范，每个 SKILL.md 包含：触发关键词、步骤说明、输入输出示例。

## 九、非功能性需求

### 9.1 错误处理

| 失败场景 | 处理 |
|---------|------|
| Memory Gateway 未启动 | dashboard 显示降级提示，MCP 工具返回错误 |
| LLM API Key 失效 | 同步脚本写错误日志，继续下次重试 |
| 单条同步失败 | 跳过 + 记日志，不影响整体 |
| 同步中断 | sync-state.json 记录进度，断点续传 |

### 9.2 性能

| 关注点 | 策略 |
|--------|------|
| 首次 Bootstrap | 异步执行 + 进度日志 |
| 每晚同步 | 按日期分批，每批最多 100 条；Token 预算上限提前停 |
| dashboard 查询 | SWR 缓存 30 秒 |
| MCP 工具响应 | 10 秒 timeout |

### 9.3 可观测性

```
~/.dream-memory/
├── gateway.log          ← Hermes Gateway 日志
├── sync-state.json      ← 同步进度
├── sync-errors.log      ← 同步错误
└── stats.json           ← L0/L1/L2/L3 各层条数、Token 消耗
```

dashboard 提供 `/api/memory/health` 暴露服务状态。

### 9.4 数据备份与迁移

- `~/.dream-memory/data/` 整个目录可打包备份
- 提供 `scripts/export.ts` 导出 JSON
- 提供 `scripts/import.ts` 从 JSON 恢复
- 跨设备同步：用户自选 iCloud / rsync / Syncthing

## 十、验证策略

### 10.1 每阶段验收标准

| 阶段 | 通过标准 |
|------|---------|
| 阶段 1 | `./start.sh` 后 `curl localhost:8420/health` 返回 200 |

### 10.2 判断点（不强行推进）

每阶段结束设观察期：

- **阶段 2 后**：用 1 周，看是否真在对话中调用 capture。0 调用 → Skill 设计有问题
- **阶段 3 后**：用 1 周，看自动同步是否有价值。都是噪音 → 同步范围/Prompt 需调整
- **阶段 4 后**：用 2 周，看画像是否准确。不准 → 升华 Prompt 需调整

前一个阶段没真正用起来，后一个阶段做了也是空壳。
