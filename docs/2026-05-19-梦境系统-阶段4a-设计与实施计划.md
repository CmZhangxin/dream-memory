## 3. 后端 API 设计

### 3.1 `GET /api/persona`

**响应**：

```ts
interface PersonaSnapshot {
  generatedAt: string;          // ISO
  totals: {
    l0Conversations: number;     // L0 行数
    l1Atoms: number;             // L1 行数（用于"升华就绪"指标）
    sessions: number;            // distinct session_key 数
    daysSpan: number;            // 第一条到最新一条跨度
    sourceFiles: number;         // distinct [VAULT:path] 数
  };
  topicDistribution: Array<{    // Section 2 饼图
    folder: string;              // 顶层目录如 "00-收件箱"
    count: number;
    pct: number;
  }>;
  dailyTimeline: Array<{        // Section 3 时间轴
    date: string;                // "2026-05-13"
    count: number;
  }>;
  topKeywords: Array<{          // Section 4 关键词
    word: string;
    freq: number;
  }>;
  health: {                     // Section 5
    hermesOk: boolean;
    hermesUptime: number;        // seconds
    lastCaptureAt: string | null;
    l1Status: "ok" | "empty" | "unknown";
  };
}
```

### 3.2 实现要点

- **数据来源**：直接 `better-sqlite3` 打开 `~/.memory-tencentdb/memory-tdai/vectors.db`（**只读模式**）
- **不调 Hermes /search**：搜不准且慢，直接 SQL 聚合
- **关键词**：用 `@node-rs/jieba`（Hermes 已装，dashboard 也加同款依赖）做中文分词 + TF 统计
- **缓存**：响应 `Cache-Control: no-store`（数据量小，每次实时跑）
- **错误处理**：SQLite 不可读 → 返回降级响应（totals=0，health.hermesOk=false）

### Task 3：API Route

**Files**:
- Create: `dream-dashboard/src/app/api/persona/route.ts`

**Code**:
```ts
import { NextResponse } from "next/server";
import { aggregatePersona } from "@/lib/persona/aggregator";

export const runtime = "nodejs";  // better-sqlite3 需要 node runtime
export const dynamic = "force-dynamic";

export async function GET() {
  try {
    const snapshot = await aggregatePersona();
    return NextResponse.json(snapshot, {
      headers: { "Cache-Control": "no-store" },
    });
  } catch (err) {
    return NextResponse.json(
      { error: err instanceof Error ? err.message : "unknown" },
      { status: 500 },
    );
  }
}
```

## 5. 风险

| 风险 | 影响 | 缓解 |
|---|---|---|
| `@node-rs/jieba` 在 Next.js 16 server runtime 加载失败 | 关键词云空白 | 加载失败时降级用简单空格分词；记 issue |

| 数据量增大后聚合慢（>1 万条）| 当前 1513 OK，未来可能 | 暂不缓存，遇到再说；YAGNI |
| L1=0 显示太刺眼 | 用户体验差 | health-card 用"已知 issue"友好措辞，链接到诊断文档 |
