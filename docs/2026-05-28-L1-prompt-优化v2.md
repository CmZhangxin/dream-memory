# L1 抽取 Prompt 优化（v2）

> 2026-05-28 改造记录

## 背景

业界主流 AI 记忆系统（Mem0/Letta/Zep）调研结论：
- **Mem0** 用 7 类信息（preferences/details/plans/health/relationships/work/activities），LOCOMO 综合最优
- **Letta**（MemGPT）OS 分页式
- **Zep** 时间感知图谱 + 边失效

dream-memory 当前架构对比业界：
- ✅ **已具备**：两阶段抽取（extraction + dedup）、4 种 action（store/skip/update/merge），与 Mem0 ADD/NONE/UPDATE/DELETE 等价
- ❌ **缺失**：抽取 prompt 类型清单太窄（仅 persona/episodic/instruction 三类）、"宁缺毋滥"过滤过狠，导致工作决策、项目事实、踩坑教训都被丢弃

## 改造内容

文件：`~/.dream-memory/tdai-memory-openclaw-plugin/src/core/prompts/l1-extraction.ts`

### 1. 拓宽 persona 子类（参考 Mem0）

新增：
- personal_details（姓名/职业/居住地）
- preferences（技术栈/UI/工具）
- skills（擅长语言/框架）
- habits（工作流/作息）
- health（饮食禁忌）
- relationships（团队/家庭）
- constraints（硬性限制：不用 X / 不接受 Y）

### 2. 拓宽 episodic 子类

新增明确分类：
- **decision**：用户做出的明确决策（高优先级 priority>=85）
- **plan**：长期计划/路线图
- **project_fact**：项目事实（仓库/分支/许可/部署位置）
- **outcome**：完成的重要事件
- **lesson**：踩坑教训/根因（高优先级 priority>=80）

### 3. 引入分层抽取

替代"宁缺毋滥"原则：
- **Tier 1（必抽）**：稳定偏好、全局指令、明确决策、踩坑教训、健康/安全
- **Tier 2（条件抽）**：项目事实、工作流约定、长期计划、领域知识
- **Tier 3（不抽）**：流水账动作、纯结果反馈、临时调试参数

### 4. 增加 6 个 few-shot 示例

涵盖：决策类、项目事实、踩坑教训、流水账（反例）、偏好、临时指令（反例）

### 5. 显式触发词列表

每个类型给出明确触发词，降低 LLM 漏抽概率：
- persona："喜欢、习惯、不希望、坚持用、我这个人..."
- episodic："决定、敲定、用 X 不用 Y、踩坑、根因是、托管在..."
- instruction："以后都、从现在开始、必须、不要、回复要..."

## 预期效果

之前 L1 7 条全是"用 antd""用 cron"这类泛偏好。改造后预期能抽到：
- ✅ "用户决定 LLM 用 SiliconFlow + DeepSeek-V3，理由是 OpenAI 贵..."
- ✅ "项目托管在 GitHub xxx，主分支 main，许可证 MIT"
- ✅ "踩坑：cp -r 嵌套目录的修复方法"

## 不变项（已具备的能力）

- 两阶段管道：extraction → dedup（store/skip/update/merge）
- 4 种 action 等价于 Mem0 ADD/NONE/UPDATE/DELETE
- 跨 type 合并（persona/episodic/instruction 互转）
- Top-K 向量召回 + 全局候选池
