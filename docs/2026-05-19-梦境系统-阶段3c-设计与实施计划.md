# 梦境系统 · 阶段 3c 设计与实施计划：Skills 编排（B 方案）

**版本**：v1.0
**日期**：2026-05-19
**前置**：阶段 2 (`dream-memory-mcp`) + 阶段 3b (cron 自动同步) 已完成
**目标**：让 CodeBuddy 对话能**主动调用** dream_* 工具捕获决策与日终总结

---

## 1. 范围（B 方案）

### 1.1 做

- 把 `dream-memory-mcp` 注册到全局 `~/.codebuddy/mcp.json`，让所有 CodeBuddy 对话都能调 12 个 `dream_*` 工具
- 写 2 个核心 Skill：`capture-decision`、`daily-review`
- 写项目级 `CODEBUDDY.md`，引用 Skills 并声明使用约定（兜底，确保即使 Skill 自动加载机制不工作也能用）

### 1.2 不做（YAGNI / 留观察）

- ❌ `dream-now` Skill（升华命令）—— `dream_session_end` 已经能直接调，不需要专门 Skill 包装
- ❌ Slash Command（如 `/capture`）—— 直接说话就能触发，不需要前缀
- ❌ Skill 自带子工作流（如自动归档）—— 先看真实使用习惯
- ❌ 修改 `dream-memory-mcp` —— 阶段 3c 不动 MCP 服务

### 1.3 设计决策

| # | 决策 | 选择 |
|---|------|------|
| 1 | 范围 | B（mcp 接入 + 2 Skill）|
| 2 | 接入位置 | 全局 `~/.codebuddy/mcp.json` |
| 3 | Transport | stdio（不依赖 Hermes 是否在跑 —— 真要 capture 时若 Hermes 没起，工具会返回错误，正好提醒你 `./start.sh`）|
| 4 | Skill 加载机制 | 双保险：项目级 `.codebuddy/skills/<name>/SKILL.md` + 项目根 `CODEBUDDY.md` 引用（前者依赖 CodeBuddy 自动扫描，后者 100% 加载）|
| 5 | Skill 触发关键词 | `capture-decision`：决策/选择/拍板/敲定 / `daily-review`：日终/复盘/今天总结/今天做了什么 |

## 2. 架构

```
[你说: "今天决定先接 MCP 不写 Skill"]
        ↓
[CodeBuddy Agent (Claude)]
        ↓ 命中关键词触发 capture-decision Skill
[SKILL.md 被注入 system prompt：调用 dream_capture_decision]
        ↓
[dream-memory-mcp via stdio]
        ↓ HTTP /capture
[Hermes Memory Gateway :8420]
        ↓
[Hermes 升华: L0 → L1 → L2 → L3]
```

**关键**：Skill 是**给 LLM 看的操作手册**，不是代码。它定义"在什么对话信号下该调什么工具、参数怎么填"。

### Task 2：写 capture-decision Skill

**Files**:
- Create: `.codebuddy/skills/capture-decision/SKILL.md`

**Skill 内容设计要点**：
- frontmatter `name: capture-decision`
- description 写明触发条件：用户说"决定 / 拍板 / 敲定 / 选择了 X / 排除 Y"
- 操作手册：
  - 1) 总结决策核心（一句话）
  - 2) 列举备选方案与排除原因
  - 3) 调用 `dream_capture_decision`，参数：
    - `text`: 决策摘要
    - `metadata.alternatives`: 排除的备选
    - `metadata.context`: 上下文链接（任务/PR）
  - 4) 简短 ack（"已记录"）
- 避免：每次 Agent 说什么都触发；只在用户**明确**说决策时

### Task 3：写 daily-review Skill

**Files**:
- Create: `.codebuddy/skills/daily-review/SKILL.md`

**Skill 内容设计要点**：
- description 写明触发条件：用户说"日终 / 复盘 / 今天做了什么 / 总结今天"
- 操作手册：
  - 1) 调用 `dream_recall`，query=今天的关键词，session_key=今日
  - 2) 让 Agent 整理成 3-5 条 bullet
  - 3) 让用户确认/补充
  - 4) 调用 `dream_capture_note`，metadata.tag=daily-review
  - 5) 询问是否触发升华（`dream_session_end`）

## 4. 风险

| 风险 | 影响 | 缓解 |
|---|---|---|
| 项目级 `.codebuddy/skills/` 不被 CodeBuddy 自动加载 | Skills 无效 | 双保险：CODEBUDDY.md 直接引用内容 |
| Skill 关键词触发太频繁 | 信号噪声 | description 写明"用户**明确**表达"才触发；不是任何包含"决定"二字都触发 |
| Hermes 没跑时 capture 失败 | 用户困惑 | Skill 在工具调用失败后，提示 `cd dream-dashboard && ./start.sh` |
