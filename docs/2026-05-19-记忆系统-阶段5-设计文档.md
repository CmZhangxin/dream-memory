## 一、核心定位

**记忆系统 = 让所有 AI 工具共享一份关于"你"的记忆，越用越懂你。**

不是画像展示，不是笔记管理，不是知识库。
是一层**中间件**，让 CodeBuddy / Claude / ChatGPT 都能：
- 写入记忆（AI 自动捕获你的偏好、决策、反馈）
- 读取记忆（AI 启动时自动知道"你是谁、你喜欢什么、之前决定了什么"）

### 四层记忆架构

```
┌─────────────────────────────────────────────────────────┐
│  L3 · Persona（服务画像）                                 │
│  稳定的用户偏好与服务方式画像                               │
│  → 让 Agent 按用户习惯协作                                │
│  → 存储：persona.md（单文件 Markdown）                    │
│  → 触发：累计 50 条新 L1 后自动重新生成                    │
│  → 注入位置：system prompt 末尾 <user-persona> 标签       │
├─────────────────────────────────────────────────────────┤
│  L2 · Scene Block（场景块）                               │
│  按项目/主题/工作流场景聚类                                 │
│  → 存储：scene_blocks/*.md + .metadata/scene_index.json  │
│  → 触发：L1 完成后 90 秒触发，最小间隔 15 分钟             │
│  → LLM 自主决定 CREATE/UPDATE/MERGE/DELETE 场景           │
│  → 注入位置：system prompt 末尾 <scene-navigation> 标签   │
├─────────────────────────────────────────────────────────┤
│  L1 · Atomic Memory（原子记忆）                           │
│  自动提取事实、偏好、约束、状态                              │
│  → 存储：SQLite vectors.db l1_records 表 + 向量索引       │
│  → 触发：每 5 条对话一批（warmup 从 1 开始翻倍）           │
│  → 注入位置：user prompt 前缀 <relevant-memories> 标签    │
├─────────────────────────────────────────────────────────┤
│  L0 · Raw Log（原始日志）                                 │
│  全量保留原始对话与事件流                                   │
│  → 存储：SQLite vectors.db l0_conversations 表            │
│  → 证据兜底，升华的原材料                                  │
└─────────────────────────────────────────────────────────┘

数据流：碎片对话 → 结构事实 → 场景认知 → 服务画像
```

### Pipeline 调度参数（默认值）

| 参数 | 默认值 | 含义 |
|---|---|---|
| `everyNConversations` | 5 | 每 5 条对话触发一次 L1 batch |
| `l1IdleTimeoutSeconds` | 600 | 10 分钟空闲也触发 L1 |
| `l2DelayAfterL1Seconds` | 90 | L1 完成后 90s 触发 L2 |
| `l2MinIntervalSeconds` | 900 | L2 最小间隔 15 分钟 |
| `l2MaxIntervalSeconds` | 3600 | L2 最大间隔 60 分钟 |
| `persona.triggerEveryN` | 50 | 累计 50 条新 L1 后触发 L3 |

### Recall 注入协议（已实现）

```typescript
interface RecallResult {
  prependContext?: string;        // L1 相关记忆 → user prompt 前缀
  appendSystemContext?: string;   // L3 Persona + L2 导航 → system prompt 末尾
  recalledL1Memories?: RecalledMemory[];
  recalledL3Persona?: string | null;
}
```

注入格式：
```
[system prompt 末尾]
<user-persona>
用户是 AI 工程师，偏好 antd + TS strict...
</user-persona>

<scene-navigation>
| 场景 | 摘要 | 热度 |
| workshop | 记忆系统建设... | 12 |
| mall | 电商项目... | 8 |
</scene-navigation>

[user prompt 前缀]
<relevant-memories>
以下是当前对话召回的相关记忆：
- 喜欢 antd 不用 MUI
- workshop 项目决定用 cron
- 别太啰嗦
</relevant-memories>
```

## 三、6 个痛点的解法映射

| 痛点 | 哪层解决 | 机制 |
|---|---|---|
| A 重复说明偏好 | L1 提取偏好 + L3 注入 Agent | AI 启动时 appendSystemContext 自动带 Persona |
| B 忘记决策 | L1 提取决策 + L2 按项目聚类 | 同项目决策在同一 Scene Block 里 |
| C 找不到旧对话 | L0 原始日志 + L1 结构化搜索 | `/search/conversations` + `/search/memories` |
| D 人物/项目事实 | L1 提取事实 + L2 按场景聚类 | Scene "客户A" 聚合所有相关事实 |
| E AI 方案矛盾 | L2 上下文召回 | 同一 Scene 的历史都在 navigation 里 |
| F 反馈失效 | L1 提取反馈 + L3 固化到 Persona | "别啰嗦"变成 Persona 永久条目 |

操作说明：
- **强化** = confidence + 0.2（上限 1.0），该记忆更容易被 recall 命中
- **降权** = confidence - 0.2（到 0.1 以下自动归档）
- **编辑** = 修改内容/分类，不立即触发 L2/L3 重建
- **删除** = 软删除 + L2 标脏

## 八、飞轮机制

### 自动捕获

```
AI 对话中 → dream_capture_conversation（写 L0）
         → 每 5 条对话触发 L1 batch extraction（LLM 自动提取）
         → L1 完成 90s 后触发 L2（场景聚类）
         → 累计 50 条 L1 后触发 L3（Persona 更新）
```

### 自动注入

```
每次新对话 → AI 调 dream_recall(context)
          → 返回：appendSystemContext(L3 Persona + L2 导航)
                 + prependContext(Top K L1 atoms)
          → AI 行为对齐你
```

### 人工纠偏

```
Dashboard → 审核 pending / 编辑 L1 / 强化降权 / 查看 Persona / 手动重新生成
```

### 冷启动（引导向导）

```
步骤 1/3 · "你用什么技术栈？"
  → 自动拆成 3-5 条 preference L1

步骤 2/3 · "你的工作习惯？"（多选）
  → 每选一条 = 1 条 L1

步骤 3/3 · "你正在做什么项目？"
  → 创建 L2 Scene 初始骨架 + 项目 fact L1
```

### 飞轮量化指标

| 指标 | 健康值 |
|---|---|
| L1 总数 | 30-200 条 |
| L2 场景数 | 5-20 个 |
| L3 最近更新 | < 7 天 |
| 每周新增 L1 | > 5 条/周 |
