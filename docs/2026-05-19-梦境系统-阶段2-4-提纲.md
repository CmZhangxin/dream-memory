## 阶段 2：MCP 工具集（dream-memory-mcp）

**Goal:** 新建 `dream-memory-mcp` 项目，实现 12 个 dream_* MCP 工具，并通过 `~/.codebuddy/mcp.json` 注册到 CodeBuddy。

**Architecture:** TypeScript ESM；MCP SDK + Hermes Gateway HTTP 客户端；每个工具一个文件，清晰分组（capture/get/sync）。

**Tech Stack:** TypeScript 5、@modelcontextprotocol/sdk、zod、tsx

**预计工期：** 2 天

### Tasks 提纲

- **Task 1**：初始化 `dream-memory-mcp/` 项目骨架（package.json、tsconfig、目录结构）
- **Task 2**：实现 `gateway-client.ts`（封装 Hermes Gateway HTTP 调用，含错误处理）
- **Task 3**：实现 MCP server.ts 入口 + 注册路由
- **Task 4**：写 5 个 capture 工具（capture_note / capture_task / capture_conversation / capture_skill / capture_decision）
- **Task 5**：写 5 个 get 工具（search / get_persona / get_today / get_scenarios / get_related）
- **Task 6**：写 2 个 trigger 工具（sync_source / dream_now）
- **Task 7**：单元测试（mock Gateway，验证工具行为）
- **Task 8**：写 `~/.codebuddy/mcp.json` 配置示例 + README
- **Task 9**：手工集成测试（在 CodeBuddy 对话里调用）
- **Task 10**：发布到 npm（可选，先 link 本地用）

### 验收
- CodeBuddy 对话："记下来：start.sh 端口要和 package.json 对齐" → 触发 dream_capture_note → Memory 中查到
- CodeBuddy 对话："搜索：端口冲突" → 触发 dream_search → 返回相关记录

### 依赖
- 阶段 1 已完成（Memory Gateway 必须能起来）
- 用户已配置 LLM API Key

### 待启动前细化
- MCP SDK 版本选择（@modelcontextprotocol/sdk 当前主流版本）
- gateway-client 的具体 HTTP 路径（需读 Hermes 源码确认 `/recall` `/capture` `/search/*` `/session/end` 等接口签名）
- Zod schema 设计

## 阶段 3：采集 + Skills

**Goal:** 让 Obsidian Vault 和 CodeBuddy 项目日志每晚自动同步进 Memory；让 CodeBuddy 用户能通过自然对话触发"沉淀/复盘/做梦"工作流。

**Architecture:** 两条独立的子工程同时推进：
1. `dream-dashboard/scripts/` 三个 ts 脚本 + node-cron 调度
2. `.codebuddy/skills/` 三个 SKILL.md

**Tech Stack:** TypeScript、tsx、node-cron、gray-matter（已在 dashboard 依赖中）、Markdown 解析

**预计工期：** 2-3 天

### Tasks 提纲

#### Part A：采集脚本

- **Task A1**：实现 `sync-state.ts`（读写 `~/.dream-memory/sync-state.json`）
- **Task A2**：实现 `sync-vault.ts`（扫描 Obsidian Vault，按 mtime 增量，调 dream_capture_note）
- **Task A3**：实现 `projects.yml` 加载 + slug 反查工具
- **Task A4**：实现 `sync-codebuddy.ts`（扫描白名单项目的 .codebuddy/memory/*.md + projects/<slug>/*.jsonl）
- **Task A5**：实现敏感词过滤（密码/token/邮箱/手机号正则）
- **Task A6**：实现 ignore 规则（.gitignore 语法）
- **Task A7**：实现 `daily-dream.ts`（统一入口：sync-vault + sync-codebuddy + dream_dream_now）
- **Task A8**：实现独立 cron-runner.ts（node-cron 每晚 23:00 触发）
- **Task A9**：把 cron-runner 加入 start.sh / stop.sh 生命周期管理

#### Part B：CodeBuddy Skills

- **Task B1**：创建 `.codebuddy/skills/capture-decision/SKILL.md`
- **Task B2**：创建 `.codebuddy/skills/daily-review/SKILL.md`
- **Task B3**：创建 `.codebuddy/skills/dream-now/SKILL.md`
- **Task B4**：手工对话测试三个 Skill 的触发

### 验收
- 23:00 自动跑一次同步，第二天看到新 Atom
- 在 CodeBuddy 说"复盘今天" → daily-review 触发 → 看到结构化叙事

### 依赖
- 阶段 2 完成（MCP 工具能用）
- `~/.dream-memory/projects.yml` 已配置至少 1 个项目
